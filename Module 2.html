Module 2 - Network Security

Module 2 Units 
Unit 1: History of Network Security, Vulnerabilities and Approaches
Unit 2: Advanced Persistent Threats: Applying the Cyber Kill Chain Model to a Case Study
Unit 3: Vulnerability Assessments
Unit 4: Breach Analysis and Mitigation
Unit 5: Logging, Forensics and Future Trends
Unit 6: The Great Debate: The Future of the Internet
Network Security 
This module covers the impact of Cyber Security from the perspective of Design and Implementation. 
There will be a project that focuses heavily on Network Vulnerability Assessments.
Throughout this module, I will be making notes on the content that has been covered and recording my observations here.

Unit 1:
Unit 1 was an overview of the challenges of implementing a secure Cyber Security defense.
It was clear from the reading material, that we would be working towards learning and understanding, 
how to perform a vulnerability assessment of a website as part of our module assessment strategy.
We covered in depth, the security breach implemented by Russian aggressors using Solarwinds Network Management Software. 
The fact that this breach was really only discovered by accident, is shocking, but it leads us to believe, 
that there could be many more breaches in other Software, that is lying dormant or undetected until now. 
The reputation of Solarwinds suffered greatly, but they may not be unique.
We covered the concept of the Cyber Kill Chain, as a methodology for understanding how Cyber Attacks may be implemented.


Unit 2:
Unit 2 began with the concept of Threat Modelling as a strategy, to strengthen the Security perimeter, and mitigate the risk of attack. 
Concepts such as STRIDE and DREAD were introduced, which define a type of Security risk, and then, 
if discovered, allocate a “cost” to it based on its severity. 
As part of my reading, I discovered that Microsoft, who effectively created the DREAD classification model, 
have actually started moving away from this, and have simplified their classification strategy.
This unit also covered the basic requirements of any good IT Infrastructure that are required for a good Cyber defense.
Network Penetration Testing was also introduced, starting with the concepts, 
and then moving on to the possible methodologies and tools that can be utilized.
We had been allocated a particular website for test purposes, and started to examine its security with Penetration Testing Tools.
The move for businesses to digitized platforms was covered, and included the risk vs benefit of the migration, as well as the ongoing support required.


Unit 3:
Unit 3 increased the focus on Penetration Testing, and the methodology that can be used to assess it.
Initial scans of the website allocated to us were performed, both to establish an initial assessment of the current Security implementation, 
and to get practice using some of the available tools.
At the end of this unit, there was an assessment to perform a baseline analysis of the website allocated to us, 
and put it into a report that would show a complete plan for a full assessment.
Web tools were used predominantly here.


Unit 4:
Unit 4 continued with the focus on Penetration testing as a tool for Network assessment, but it introduced an Open-Source tool called Kali Linux. 
Kali is a mostly free Penetration toolkit, based on a Linux image. 
It can be installed directly onto a partition, or, as I did it, run through some virtualization software called VirtualBox, 
and install the .ISO from there.
Kali tools are much more useful and powerful than those typically available for free on the Public Internet, 
and it allowed me to perform some tests that I was unable to before, and with more information returned.
The use of logging on devices as a means for attackers to gain access to systems was also discussed.


Unit 5:
Unit 5 continued with the theme of logging, and the risks posed. Clearly, the benefit of logging is understood, but it needs to be secured as well. 
The concept of a centralized log collector was discussed, as a “one-stop shop” that collects and collates all logs from all Infrastructure devices. 
This tool is then supposed to be able to combine the different log events to a particular incident, 
which is then highlighted in a useful way in the tool. 
Work continued on the end of module assessment, that focuses on the actual Penetration Test analysis of the website that was allocated to us. 
We were provided with access to sample reports, and others were found online, but all had no 1200-word restriction assigned to them.


Unit 6:
Unit 6 is the final week in the Module, and the main focus is the end of module assessment, in the form of the Website Vulnerability Security Assessment. 
The report was compiled predominantly using tools from Kali Linux, but some website tools were also used, 
as the various options complimented each other in some cases. 
There were, however, times where Kali was the only tool I could find for some tests.
In the required reding, there is emphasis on the Future of the Internet, and what it holds, 
as well as the obvious difficulties in how to secure it.
Finally, there is a Reflective Piece to be written, covering what was learned over the course of the module.
